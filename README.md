# socratic-study

一个适用于 **Claude Code** 的全局学习 skill。  
它会围绕你当前目录中的知识文件，用**苏格拉底式问答**带你学习，并把每轮学习记录写回当前目录，方便复盘。

---

## 1. 这个 skill 是做什么的

这个 skill 适合这样的场景：

- 你有一批知识文件放在某个文件夹里
- 你想直接在那个文件夹里打开 Claude Code
- 然后让 Claude Code 围绕这些资料带你学习
- 不是直接讲答案，而是通过提问、追问、提示、举例来帮助你理解
- 并且把你的学习过程记录下来

它支持：
- 在当前目录里发现知识文件
- 围绕知识文件进行苏格拉底式提问
- 用户答不上来时逐级提示
- 把记录写入当前目录的 `苏格拉底学习记录.md`

---

## 2. 支持的知识文件格式

当前默认支持识别：

- `*.md`
- `*.markdown`
- `*.txt`
- `*.pdf`
- `*.docx`

如果当前目录中有多个文件但你没有明确指定，Claude 应该先让你选择，而不是乱猜。

---

## 3. 安装方法

把仓库里的这个文件：

```text
socratic-study/SKILL.md
```

复制到你的 Claude Code 全局 skills 目录下：

### Windows
```text
C:\Users\<你的用户名>\.claude\skills\socratic-study\SKILL.md
```

### macOS / Linux
```text
~/.claude/skills/socratic-study/SKILL.md
```

如果 `socratic-study` 文件夹不存在，就自己创建。

安装完成后：
- 重启 Claude Code，或
- 重新加载 skills

---

## 4. 怎么使用

## Step 1：进入你的知识文件目录
例如：

```text
D:\study\database\
  ├─ 数据库笔记.md
  ├─ 事务.pdf
  └─ 索引.docx
```

在这个目录里打开 Claude Code。

## Step 2：调用这个 skill
你可以直接对 Claude Code 说类似：

- “请用 socratic-study 围绕当前目录里的资料带我学习。”
- “用苏格拉底法带我学当前目录中的数据库文件。”
- “我想围绕这个 PDF 开始学习。”
- “从封装开始问我问题。”

---

## 5. 学习过程中你可以怎么说

### 开始
- “开始学习”
- “围绕这个文件开始问我问题”
- “从事务开始”

### 回答
- “我的回答是……”
- “我觉得它的意思是……”

### 请求帮助
- “给我一点提示”
- “我答不上来”
- “举个例子”
- “换个更小的问题问我”

### 结束
- “结束这次学习并总结”
- “帮我做本次回顾”

---

## 6. 学习记录会保存到哪里

默认会写到当前目录下：

```text
./苏格拉底学习记录.md
```

例如，如果你在这个目录里使用：

```text
D:\study\database\
```

那么记录会出现在：

```text
D:\study\database\苏格拉底学习记录.md
```

如果文件不存在，会自动创建。  
如果已经存在，会继续追加，不覆盖旧记录。

---

## 7. 学习记录长什么样

大致格式类似：

```markdown
# 苏格拉底学习记录

## 会话：2026-04-02 14:30
- 主题：事务
- 来源文件：database.md

### 第 1 轮
- 问题：事务是什么？
- 我的回答：……
- 反馈：……
- 下一步：继续追问 / 提示 / 举例

### 第 2 轮
- 问题：为什么事务需要隔离性？
- 我的回答：……
- 反馈：……
- 下一步：……

### 本次小结
- 已掌握：……
- 部分掌握：……
- 仍需复习：……
```

## 9. 使用注意事项

### 1. 要在有知识文件的目录里使用
如果当前目录没有知识文件，skill 无法围绕资料学习。

### 2. 多文件时最好明确指定
如果当前目录里有很多文件，最好告诉 Claude：
- 先围绕哪个文件
- 或者先围绕哪个主题

### 3. 它不是后台自动程序
这是一个 Claude Code skill，不是 daemon。  
它会在你调用时工作，并用 Claude Code 的现有工具读文件、问答、写记录。

### 4. 默认只写当前目录
除非你明确要求，否则它不会把记录写到当前目录之外。

- “继续问我”
